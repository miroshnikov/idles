IDLES PHP functional library

IDLES - A PHP FUNCTIONAL UTILITY LIBRARY

What is it?

Idles library may be seen as a port of Javascript Lodash and Ramda libraries to PHP, as it tries to comply with Lodash and Ramda APIs as much as possible. Though there are inevitable discrepancies in methods API considering the different nature of PHP and JavaScript languages.

Requirements

PHP: ^7.4

Installation

composer require miroshnikov/idles

Usage

All methods are in Idles\ namespace, immutable and auto-curried. 

use function Idles\{pipe, map, divide, ary, partial, curryN, filter};
use const \Idles\_;
pipe(
    map(divide(_, 2)),
    map(ary(1, partial('\round', [_, 0]))),
    map(curryN(1, '\intval')),
    filter(fn ($n) => $n % 2)
)([2,3,4,8]);

Features

Most of the functions are lazy (idle) when given an Iterator or Generator as an input. Such function will not load an iterator into memory, but returns a new iterator that modifies the input iterator. This behaviour is similar to iterator adaptors in Rust language or LINQ in C#.

Although when given an array such method will NOT be lazy and return a modified array. The rule is: you will get back the same kind of iterable that you passed in as an argument.

collect function may be used to turn an iterable of any kind into an array. 

use function Idles\{pipe, map, filter, collect};

$generator = (function () {
    $fp = \fopen('/huge/file/with/numbers', 'r');
    while (($s = \fgets($fp)) !== false) {
        yield $s;
    }
})();

$iterator = pipe(
    map(fn ($v) => \intval(\trim($v), 10)),
    filter(fn ($v) => $v % 2 == 0),
    map(fn ($v) => "[$v]")
    // ...
)($generator);
// nothing has happened so far, $generator is not consumed

// now you can iterate over elements of $iterator with foreach
// or convert it into an array in memory by iterator_to_array or collect
$array = collect($iterator);

The comparison everywhere is a weak equality check (==).

Binary safe, mb_* functions are used for string methods.

Array vs Record

The Idles library makes a distinction between arrays and records.

Array has only numerical sequential indexes. This is what the Array is in JavaScript and many other languages.

Record may have both numerical and string keys. AKA hash, map, object, hash table, or associative array in other languages. This is what the Object is in JavaScript.

Methods from Array section do not retain string keys and always return indexed iterables.
